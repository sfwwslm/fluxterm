# 设置交互规范（ConfigModal + ProfileModal）

## 目标

统一设置类模态框的提交策略与反馈机制，降低不同保存模式带来的认知差异。

## 字段分级

- L1 即时项：开关、下拉、单选
  交互：变更即生效，采用自动保存
- L2 普通文本项：低风险文本或数值
  交互：允许草稿输入，失焦或按 Enter 提交到状态层，再由持久化层防抖保存
- L3 高风险项：API Key、Base URL、模型标识、外部路径等
  交互：必须显式点击“保存”或按 Enter 才提交

## 保存反馈

- 自动保存项统一展示 `保存中`、`已保存`、`保存失败`
- 失败状态必须可见，并提供“重试”
- 错误细节可显示简短文本，不要求依赖日志进行基本判断

## 离开保护

- 当存在 L3 未保存草稿时，关闭模态框必须进行二次确认
- 关闭确认仅表示放弃未保存草稿，不回滚已保存内容

## 覆盖范围

- `ConfigModal`
- `ProfileModal`

## 约束

- 优先保证交互一致性与错误可见性
- 不引入复杂的批量提交流程
- 新增设置项需完成 L1/L2/L3 分级后再确定交互方式
