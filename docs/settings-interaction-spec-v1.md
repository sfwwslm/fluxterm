# 设置交互规范 v1（ConfigModal + ProfileModal）

## 目标

统一设置类模态框的提交策略与反馈机制，避免“有的自动保存、有的显式保存”造成的心智负担。

## 字段分级

- L1 即时项：开关、下拉、单选。
  - 交互：变更即生效，走自动保存。
- L2 普通文本项：低风险文本/数值。
  - 交互：允许草稿输入，失焦或回车提交到状态层，再由持久化层防抖保存。
- L3 高风险项：API Key、Base URL、模型标识、外部路径等。
  - 交互：必须显式点击“保存”或按 Enter 才提交。

## 保存反馈

- 自动保存项统一展示状态：`保存中`、`已保存`、`保存失败`。
- 失败状态必须可见，并提供“重试”。
- 错误细节可以显示简短错误文本，不要求用户打开日志排查。

## 离开保护

- 当存在 L3 未保存草稿时，关闭模态框必须二次确认。
- 关闭确认仅表示放弃未保存草稿，不回滚已保存内容。

## v1 覆盖范围

- `ConfigModal`
- `ProfileModal`

## v1 约束

- 先保证交互一致与错误可见，不在 v1 引入复杂批量提交流程。
- 后续新增设置项必须按 L1/L2/L3 分级后再落地交互。
