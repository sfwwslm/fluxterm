# FluxTerm 路线图

本文档用于记录后续计划中的主要能力方向和大致优先级，同时保留近期已落地的重要能力，便于对照产品演进方向。

## 近期已完成

### 远程文件编辑

已交付能力：

- 支持从文件管理器直接打开远程文件
- 统一复用系统默认程序或用户配置的默认编辑器
- 远端文件会下载到应用受管缓存目录后再打开
- 文本类文件支持“保存后确认回传”闭环
- 回传前会检查远端文件是否已发生变化，避免静默覆盖
- 非文本文件当前仅支持拉起本地程序打开

后续优化：

- 补充单独设计文档，固化缓存目录、轮询检测和冲突处理策略
- 根据体验再决定是否增加可见的“远端编辑会话”列表

## 当前规划

### P1 SSH Profile 高级连接能力

目标：面向所有 SSH Profile 提供更完整的连接配置能力，让手动添加和导入生成的配置在连接行为上保持一致，逐步支持跳板机、代理、主机密钥校验和多密钥等高级能力。

下一步：

- 明确哪些高级字段需要纳入 SSH Profile 编辑界面
- 梳理高级字段从 Profile 到连接链路的传递与生效方式
- 明确 `ProxyJump`、`ProxyCommand`、多 `IdentityFile`、`UserKnownHostsFile`、`StrictHostKeyChecking`、`AddKeysToAgent` 的分阶段支持范围
- 补一份单独设计文档

### P2 串口功能

目标：基于 `tokio-serial` 增加串口连接能力，支持本地串口设备调试和终端交互。

下一步：

- 明确串口连接配置项
- 评估如何复用现有终端能力
- 补一份单独设计文档

### P3 远程桌面功能

目标：基于 `IronRDP` 增加远程桌面访问能力，扩展 FluxTerm 的远程运维场景。

下一步：

- 评估窗口承载方式
- 评估接入范围和实现复杂度
- 补一份单独设计文档
