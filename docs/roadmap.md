# FluxTerm 路线图

本文档用于记录后续计划中的主要能力方向和大致优先级。

## 当前规划

### P1 远程文件编辑

目标：从文件管理器直接打开远程文件，并在应用内完成编辑和保存，方便修改远程设备上的配置文件、脚本和文本文件。

下一步：

- 明确远程文件编辑界面的承载方式
- 明确打开、修改、保存的基本流程
- 补一份单独设计文档

### P2 串口功能

目标：基于 `tokio-serial` 增加串口连接能力，支持本地串口设备调试和终端交互。

下一步：

- 明确串口连接配置项
- 评估如何复用现有终端能力
- 补一份单独设计文档

### P3 远程桌面功能

目标：基于 `IronRDP` 增加远程桌面访问能力，扩展 FluxTerm 的远程运维场景。

下一步：

- 评估窗口承载方式
- 评估接入范围和实现复杂度
- 补一份单独设计文档
