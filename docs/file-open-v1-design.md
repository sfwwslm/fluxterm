# 文件打开能力设计

## 目标

为文件管理器提供统一的文件打开能力，覆盖两类场景：

1. 本地 shell 文件管理器中双击普通文件时，在本机直接打开文件。
2. SSH 文件管理器中双击普通文件时，先下载到本地缓存，再复用本地打开流程。

## 能力范围

### 支持能力

1. 本地普通文件双击打开。
2. 远端普通文件下载到本地缓存后打开。
3. 提供全局默认编辑器配置。
4. 当默认编辑器不可用时，回退到系统默认程序打开。
5. 打开失败时显示横幅提示并记录日志。

### 能力边界

1. 文档仅定义“打开文件”链路，不包含远端文件修改后的同步策略。
2. 不在该能力内定义远端文件冲突处理。
3. 不在该能力内定义拖拽打开或批量打开行为。
4. 不针对二进制文件和文本文件设计差异化策略。
5. 缓存仅提供基础清理机制，不承担精细化生命周期管理。

## 交互语义

### 本地文件管理器

1. 双击目录：继续进入目录。
2. 双击普通文件：打开文件。
3. 单击仍然只负责选中，不改变现有桌面式交互语义。

### SSH 文件管理器

1. 双击目录：继续进入目录。
2. 双击普通文件：
   - 先下载到本地缓存目录。
   - 下载成功后使用本地打开能力打开。
3. 下载失败或打开失败时，显示横幅提示。

## 默认编辑器配置

### 配置位置

默认编辑器属于本机行为，不属于会话行为，因此配置应放在“设置”中，而不是“会话设置”。

### 配置键

- `file.defaultEditorPath`

### 打开优先级

双击打开普通文件时，优先级如下：

1. 如果配置了默认编辑器路径，并且该程序存在且可启动，则优先使用该编辑器打开。
2. 如果默认编辑器未配置或启动失败，则回退到系统默认程序打开。
3. 如果两者都失败，则显示失败横幅并记录错误日志。

### 配置交互

当前交互采用“选择程序”而不是手动输入路径：

1. 用户点击“选择程序”选择可执行文件。
2. 用户点击“清空”后，回退到系统默认程序打开。
3. 配置值持久化到 `file.defaultEditorPath`。

## 本地打开实现

### 前端职责

前端负责：

1. 判断双击目标是否为普通文件。
2. 调用统一的“打开文件”命令或封装。
3. 在失败时显示横幅。

### 后端职责

后端负责：

1. 根据配置决定是否优先使用默认编辑器。
2. 校验默认编辑器路径是否存在。
3. 执行打开程序。
4. 在默认编辑器不可用时回退到系统默认程序。

## 远端文件打开实现

### 基本流程

1. 用户在 SSH 文件管理器中双击普通文件。
2. 后端将远端文件下载到本地缓存目录。
3. 下载完成后复用本地打开逻辑打开缓存文件。

### 浮动文件面板

浮动文件面板中的“双击打开文件”统一代理回主窗口处理，不直接在浮动窗口内执行。

这样可以保证：

1. 主窗口与浮动窗口行为一致。
2. 文件打开能力只维护一套事实源。
3. 浮动窗口不持有独立的文件打开状态机。

### 缓存目录

缓存目录位于应用数据目录下，例如：

- `appData/cache/remote-files/`

### 缓存存储结构

缓存采用实例目录隔离：

- `appData/cache/remote-files/{sessionId}/{instanceId}/{fileName}`

其中：

1. `sessionId` 用于隔离不同 SSH 会话来源。
2. `instanceId` 用于隔离每次远端文件打开实例。
3. `fileName` 保持原始文件名，不做 hash 前缀污染。

### 设计原因

采用实例目录隔离，而不是直接复用同一个缓存文件，原因如下：

1. 编辑器标题和最近文件列表中应保留原始文件名。
2. 重复打开同一个远端文件时，应被视为新的本地文件实例。
3. 避免覆盖同一个缓存文件后，只触发编辑器对旧文件的热重载，而不是打开新的文件实例。

### 实例目录命名

实例目录由以下两部分组成：

1. 时间戳。
2. 短随机后缀。

示例：

- `20260301-201530-ng7k3f`

### 元数据文件

每个实例目录中额外写入一份元数据文件：

- `.fluxterm-remote.json`

建议包含：

1. `sessionId`
2. `remotePath`
3. `fileName`
4. `downloadedAt`

这份元数据文件用于问题排查、缓存管理和能力扩展。

## 缓存清理策略

缓存目录提供基础清理规则，避免长期膨胀。

### 清理原则

1. 应用启动时保留缓存目录，不做全量清空。
2. 每次打开远端文件都创建新的实例目录，不覆盖旧缓存副本。
3. 会话关闭时不立即删除缓存文件，避免外部编辑器仍在占用文件导致失败。
4. 每天首次打开远端文件时，清理超过保留期限的旧实例目录。

### 清理时机

1. 仅在“当天首次打开远端文件”前执行一次缓存清理。
2. 同一天内再次打开远端文件时，不重复清理。
3. 清理失败只记录日志，不阻断当前文件打开流程。

### 保留期限

建议策略：

1. 仅清理最后修改时间早于 7 天的实例目录。
2. 仅清理远端文件缓存目录，不影响其他应用数据。

### 非目标

缓存清理不承担以下职责：

1. 按缓存总大小限制清理。
2. 按会话维度做精细清理。
3. 强制删除正在被外部编辑器占用的文件。
4. 对单个缓存文件做独立生命周期编排。

## 状态与提示

### 成功打开

成功打开时无需额外提示，保持轻量行为。

### 失败提示

统一使用顶部横幅提示，不使用确认弹窗。

建议区分：

1. 下载远端文件失败。
2. 默认编辑器启动失败。
3. 系统默认程序打开失败。

### 日志

使用 Tauri 日志插件记录结构化日志，不使用 `console`。

建议事件：

1. `file:open-failed`
2. `remote-file:cache-download-failed`
3. `file:open-editor-fallback`

## 与文件管理器交互的关系

文件打开能力只扩展普通文件双击行为，不改变已有目录行为：

1. 目录：双击进入。
2. 文件：双击打开。
3. 单击：选中。
4. 右键：先选中再弹出菜单。
