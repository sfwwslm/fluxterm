# Changelog

本文档记录 FluxTerm 各版本已发布的用户可见更新。

格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，并遵循 [Semantic Versioning](https://semver.org/lang/zh-CN/)。

## [0.2.0] - 2026-04-25

本次更新主要聚焦于 SSH 会话个性化、状态栏资源监控信息强化、RDP 运行状态可视化，以及若干 macOS 与远程桌面细节修复。

### Added

- **SSH 会话自定义图标**: 支持为 SSH 会话配置自定义图标，方便在多连接场景下快速识别不同主机。
- **RDP 运行状态面板**: 新增 RDP 运行状态面板，可更直观地查看当前远程桌面的运行信息。

### Changed

- **状态栏资源监控增强**: 优化状态栏资源监控展示方式，并加入动态告警颜色，便于更快识别异常负载。
- **RDP 底部状态栏精简**: 简化 RDP 底部状态展示，减少无关信息对远程桌面操作的干扰。
- **前端依赖升级**: 升级 TypeScript、Vite 与相关前端依赖，为后续功能开发和兼容性维护提供更稳定基础。

### Fixed

- **macOS 配置窗口尺寸一致性**: 修复 macOS 下配置窗口尺寸不一致的问题，减少界面跳变。
- **macOS 中文输入重复字符**: 修复 macOS 中文输入法场景下按下 `CapsLock` 可能产生重复字符的问题。
- **RDP 剪贴板异常兜底**: 为 RDP 剪贴板异常补充诊断点和失败兜底状态，降低异常场景下的状态不明确问题。

### Documentation

- **许可证补充**: 补充 Apache 与 MIT 许可证文件，完善项目发布与使用说明。

## [0.1.1] - 2026-04-18

本次更新聚焦于手动更新反馈、终端自动补全体验、RDP 全屏行为，以及若干界面文案与样式细节修复。

### Added

- **更新检查反馈增强**: 手动检查更新时提供更明确的状态反馈，减少等待过程中的不确定感。

### Changed

- **代理子应用标题整理**: 简化代理子应用标题结构，提升界面信息密度与一致性。
- **多语言文案统一**: 统一中英文菜单与 RDP 相关标题命名，减少术语不一致带来的理解成本。

### Fixed

- **终端自动补全首字符偏移**: 修复首次输入字符后自动补全面板定位偏移的问题。
- **自动补全滚动条样式统一**: 修复自动补全面板滚动条样式不一致的问题，提升视觉一致性。
- **RDP 全屏覆盖异常**: 修复 Windows 下 RDP 全屏时未正确覆盖任务栏的问题，并同步窗口状态。
- **RDP 组件冗余样式**: 移除部分多余的内边距、边框与背景样式，减少界面干扰。

### Documentation

- **文档与预览更新**: 更新预览图片，并整理历史与自动补全相关设计文档内容。

## [0.1.0] - 2026-04-12

本次更新标志着 FluxTerm 0.1.0 正式版本的发布。在 beta.13 的基础上，进一步优化了 RDP 快捷键处理与 AI 浮窗的交互稳定性。

### Fixed

- **RDP 快捷键穿透**: 修复 RDP 窗口聚焦时无法使用浏览器原生快捷键（如刷新、缩放等）的问题。
- **AI 浮窗滚动抖动**: 修复 AI 浮动组件在快照重新订阅时可能产生的滚动位置跳变问题。

## [0.1.0-beta.13] - 2026-04-05

本次更新聚焦于 RDP 远程桌面体验完善、SSH 重连体验优化、主题预设扩展，以及若干连接稳定性与桌面交互细节修复。

### Added

- **RDP 全屏切换**: RDP 子应用新增全屏切换能力，远程桌面使用更沉浸。
- **RDP 双向纯文本剪贴板同步**: 新增 RDP 与本机之间的纯文本双向剪贴板同步，复制粘贴工作流更顺畅。
- **Catppuccin 主题预设**: 新增 Catppuccin 主题预设，并补充本地化主题名称，主题选择更丰富。
- **可取消的 SSH 连接状态**: SSH 连接过程中支持取消，连接建立中的控制反馈更明确。

### Changed

- **RDP 显示更流畅**: 改善高刷新场景下的画面流畅度，减少撕裂感。
- **RDP 连接行为更稳定**: 减少重复发起连接时可能出现的状态冲突问题。
- **确认对话框统一**: 移除原生 `window.confirm`，改为应用内统一确认对话框，交互风格更一致。
- **桌面表单体验优化**: 禁用输入框浏览器自动补全，减少桌面应用场景下的干扰。
- **SSH 兼容性提升**: 改善 SSH 连接性能，并补充证书认证相关能力。

### Fixed

- **SSH 立即重连**: 修复 SSH 自动重连倒计时期间无法立即按 Enter 重连的问题。
- **空白 RDP 窗口**: 修复从主菜单打开空 RDP 子应用的问题。
- **RDP 子应用关闭时机**: 修复最后一个 RDP 会话标签关闭后子应用未正确关闭的问题。
- **RDP 断开后错误复用旧桥接**: 修复 RDP 会话断开并切回标签后误复用旧桥接导致的假恢复问题。
- **退出时重复清理**: 修复应用退出时 RDP 重复关闭与空清理日志噪音问题。
- **OpenSSH 导入查重范围**: 修复 OpenSSH 配置导入时查重范围过宽的问题，避免不同分组间误判重复。
- **RDP 状态展示精简**: 隐藏非必要的 RDP 剪贴板同步状态展示，减少界面干扰。
- **界面样式细节**: 修复并统一 RDP 配置弹窗、配置面板和历史面板的边框与背景表现。

## [0.1.0-beta.12] - 2026-03-29

本次更新带来了 FluxTerm 的 RDP 远程桌面能力。现在可以直接在应用内管理 RDP 配置、从主界面发起连接，并在独立远程桌面窗口中承载会话，不再需要把远程终端与远程桌面拆到不同工具里使用。

### Changed

- **旧配置文件不兼容**: 由于本版本按协议拆分了 SSH 与 RDP 连接配置存储，旧版本遗留的配置文件不能直接兼容 `0.1.0-beta.12`。

## [0.1.0-beta.11] - 2026-03-28

本次更新聚焦于主题表现统一、透明度与独立窗口体验优化、配置界面结构整理，以及终端与安全相关细节修复。

### Changed

- **主题表现统一**: 进一步统一主题切换后的整体视觉表现，样式组织更一致。
- **透明度体验统一**: 统一主界面、子应用、浮窗、SFTP 组件等区域的背景透明度规则，并接入主题透明度体系，视觉表现更协调。
- **窗口启动体验优化**: 减少子应用与悬浮窗口打开时的闪烁和突兀感。
- **配置界面结构整理**: 统一配置分组与导航结构，设置项更容易查找和理解。
- **AI 配置更清晰**: 将 AI 请求超时纳入正式配置项，配置行为更明确。
- **基础样式收敛**: 统一浮层、滚动条、按钮和部分基础控件样式，减少界面细节上的割裂感。

### Fixed

- **终端聚焦行定位**: 修复终端聚焦行坐标映射误差，并提前到鼠标按下阶段处理，定位反馈更准确。
- **终端光标显示**: 修复终端流式输出过程中光标隐藏逻辑不稳定的问题。
- **断开横幅可见性**: 提升终端断开横幅在高透明度背景下的背景可见性，状态提示更容易辨认。
- **背景透明度交互**: 调整背景透明度滑杆语义为 `1% - 100%`，并精简相关提示信息，配置含义更直观。
- **安全状态展示**: 清理安全模式中的明文残留并统一状态展示，信息表达更清楚。
- **空组件占位样式**: 优化空组件占位内容的居中显示效果。

### Documentation

- **文档统一整理**: 统一多份设计与说明文档的风格，并更新与当前实现不一致的内容。

## [0.1.0-beta.10] - 2026-03-25

本次更新聚焦于远程文件编辑体验、SSH 配置导入增强、安全模式优化，以及本地 Shell 与界面细节体验改进。

### Added

- **远程文件系统编辑**: 新增基于系统编辑器的远程文件编辑工作流，支持将远程文件拉取到本地编辑，并在保存后回传服务器。
- **远程编辑冲突保护**: 远程文件保存时新增上传确认与远程变更冲突校验，降低覆盖他人修改或误上传的风险。
- **内置 SVG 壁纸**: 新增多套内置 SVG 壁纸，可直接在应用内选择使用，无需手动准备背景资源。

### Changed

- **OpenSSH 配置导入增强**: 支持导入更多 SSH 字段，复杂连接配置迁移更完整。
- **安全模式分级**: 将安全模式划分为“弱保护”和“强保护”，安全能力表达更清晰。
- **本地 PowerShell 发现优化**: Windows 下的本地 PowerShell 发现与重新扫描更方便。
- **文件管理交互统一**: 文件管理器中的新建目录和重命名改为应用内输入弹窗，交互风格更一致。
- **SSH 稳定性提升**: 改善 Windows 环境下的 SSH 使用稳定性。

### Fixed

- **文件管理器无效操作**: 修复不同会话类型下仍显示无效文件管理操作的问题，减少误操作。
- **SSH 主机身份确认弹窗**: 优化主机身份确认弹窗的布局与文案，信息展示更清晰。
- **macOS 遮盖层显示异常**: 修复 macOS 下主机身份确认遮盖层顶部漏出的问题。
- **主界面分隔线对齐**: 修复快捷栏与状态栏之间分隔线未居中的问题，底部区域视觉更稳定。

- **本地 Shell 设置更清晰**: 本地 Shell 与 SSH 配置区分更明确，后续管理更方便。
- **主密码体验完善**: 补齐主密码相关流程，凭据保护与安全设置体验更完整。

- **配置界面刷新问题**: 修复配置模态框左侧菜单在切换语言后不刷新的问题。
- **界面交互细节**: 调整终端重连横幅位置，移除底部快捷栏按钮悬浮上移动画，减少干扰感。

本次更新聚焦于关于页能力增强、终端输入体验修复与终端配置细化，并同步统一了浮窗、子应用和配置界面的标题栏与控件表现。

- **窗口标题栏统一**: 统一浮动窗口与子应用标题栏样式、右侧控件间距和头部结构，整体视觉与交互更一致。

- **终端快捷键误拦截**: 修复终端聚焦时部分按键被错误拦截的问题，改善 `vim`、`less`、`htop` 等 TUI 场景可用性。
- **关于页控制台作用域**: 修复从关于页打开控制台时对子应用窗口不生效的问题。

## [0.1.0-beta.7] - 2026-03-08

本次更新聚焦于 SSH/SFTP 连接稳定性、日志提示优化与本地 shell 参数一致性，重点修复了无 SFTP 服务场景下的输入卡顿问题。

- **本地 Shell 参数更一致**: 本地会话中的终端类型与字符集配置表现更稳定。
- **发布流程更可靠**: 避免重复创建 GitHub Release。

### Fixed

- **无 SFTP 服务输入卡顿**: 修复连接无 SFTP 服务主机（如部分 OpenWrt）时，检测阶段终端短暂无法输入的问题。
- **macOS 本地 Shell TERM**: 修复 macOS 打包版本中本地 shell 的 `TERM` 缺失问题，恢复 Delete/Fn+Delete 等按键行为一致性。

## [0.1.0-alpha.5] - 2026-03-03

本次更新重点补齐了 SSH 主机身份校验、OpenSSH 配置导入与凭据存储保护，并继续提升终端信息展示与连接稳定性。

### Added

- 新增 SSH 主机身份校验能力。
- 支持首次连接主机时确认主机身份，并在主机指纹变化时提示风险。
- 新增 SSH Host Key 校验策略，可在询问确认、严格阻断和关闭校验之间切换。
- 应用开始维护独立的 Host Key 信任记录，不与系统 `ssh` 命令的 `known_hosts` 混用。
- 新增从 OpenSSH `~/.ssh/config` 一键导入会话的能力，导入后的主机会统一归入 `OpenSSH` 分组。
- 导入支持读取常见的 `Host`、`HostName`、`Port`、`User`、`IdentityFile` 配置。
- 导入支持 `Host *` 默认值继承到后续具体主机块。

### Changed

- SSH 连接和资源监控都会经过主机身份校验流程，连接安全边界更清晰。
- 资源监控在主机未信任、探测失败或连接失败时，会给出明确状态，不再长时间停留在“资源检测中”。
- 资源监控会区分显示更具体的不可用原因，例如主机指纹未信任、远端系统不支持或资源采样失败。
- 资源监控采样时减少欢迎信息和交互式输出的干扰，结果更清晰。
- 终端路径联动对提示符的识别规则进一步收紧，普通欢迎信息和状态行不再被误判为不支持的提示符。
- 会话列表中的 OpenSSH 导入结果会以更紧凑的摘要提示展示，并忽略值为 0 的结果项。
- OpenSSH 导入时，过长的会话名称会自动裁剪，并保留统一的长度约束。
- 状态栏新增代码行统计百分比显示，便于更直观地了解当前终端输出规模。
- **连接稳定性增强**: 改善 SSH 连接、保活与传输稳定性。
- **项目标识统一**: 统一项目技术标识为 `fluxterm`，应用标识、配置目录和包名保持一致。

### Fixed

- 修复主机已信任后资源监控仍可能卡在“资源检测中”的问题。
- 修复退出 SSH 会话后重新连接时，主机身份确认流程可能重复创建会话的问题。
- 修复取消主机身份确认后，确认弹窗仍会被重复触发的问题。
- 修复资源监控连接失败或被阻断时，前端状态无法正确结束的问题。
- 修复终端路径联动将欢迎信息误判为不支持提示符并输出无意义警告的问题。

### Security

- **凭据存储保护**: 凭据统一采用加密 token 形式保存，配置文件中的密码字段不再以展开 JSON 形式保存。
- **信任记录归一**: 主机身份信任记录统一收纳到应用配置目录中，后续管理更清晰。
